Newly introduced — bugs created by the v2 fixes

NEW BUG
hsi_stream.jl — permutedims axes are wrong; spectral mixing remains
benchmarks/hsi_stream.jl — chunk processing loop
▼

The fix replaced the raw reshape with a permute+reshape, but chose the wrong permutation. With Julia's column-major storage, producing a (n_pixels, n_bands) matrix where row p contains all bands of pixel p requires putting the band axis last:

chunk shape: (n_bands, chunk_rows, chunk_cols)
Need:        (chunk_rows, chunk_cols, n_bands)  →  permutedims(chunk, (2, 3, 1))

Fix in code uses:  permutedims(chunk, (1, 3, 2))
Result shape:      (n_bands, chunk_cols, chunk_rows)  ← bands still first

After reshape(…, (n_pixels_chunk, n_bands)):
  pixel_spectra[p, b]  maps to  band = ((p-1) % n_bands) + 1
  i.e., "band b" of pixel p is actually band p-mod-n_bands of an arbitrary pixel.
Spectral vectors are still scrambled.
Fix
chunk_perm = permutedims(collect(chunk), (2, 3, 1)) # → (rows, cols, bands)
Then reshape(chunk_perm, n_pixels_chunk, n_bands) is correct in column-major.
NEW BUG
reprojection.R — UTM missing easting and southing offsets; every coordinate is wrong
benchmarks/reprojection.R — wgs84_to_utm()
▼

The new R implementation omits the false easting and southing that define the UTM coordinate system. The Julia implementation has them correctly:

# Julia (correct)
x[i] = x_utm + 500000.0
y[i] = lat[i] < 0.0 ? y_utm + 10000000.0 : y_utm

# R implementation (missing both)
x <- k0 * N * lon_r * cos(lat_r)
y <- k0 * N * (lat_r - ...)
# No +500000 or +10000000 — every UTM coordinate is ~500 km off in X and
# ~10,000 km off in Y for southern hemisphere points.
Fix
Add x <- x + 500000 and y[lat < 0] <- y[lat < 0] + 10000000 at the end of wgs84_to_utm().
NEW BUG
reprojection.R — single measurement only; no benchmark loop
benchmarks/reprojection.R — main()
▼

The new implementation takes exactly one timing measurement per transform with no warm-up and no repeat runs. Every other language benchmark uses 3–10 repeated runs with warmup:

start <- Sys.time()
merc <- wgs84_to_web_mercator(points$lat, points$lon)
elapsed <- as.numeric(difftime(Sys.time(), start, units="secs"))
# That's it — one cold run, no loop

The reported min_time_s is a single cold-run sample, incomparable with the 5–10 warm-run minimum times the other languages report.

Fix
Wrap in run_benchmark() following the same pattern as raster_algebra.R, using at least 5 runs and 2 warmup runs.
NEW BUG
reprojection.R — output JSON structure incompatible with cross-language comparison tools
benchmarks/reprojection.R, tools/compare_results.py
▼

The R output nests results under results$mercator (a single entry for n=10000), while the Julia and Python outputs key by size: mercator_1000, mercator_5000, mercator_10000. The tools/compare_results.py looks for these suffixed keys and will find nothing from R, so R is silently excluded from all speedup tables.

# R output
{"results": {"mercator": {"min_time_s": 0.001, "n_points": 10000}}}

# Julia/Python expected by compare tools
{"results": {"mercator_1000": {...}, "mercator_5000": {...}, "mercator_10000": {...}}}
Fix
Loop over sizes [1000, 5000, 10000] in the R implementation and key results as paste0("mercator_", size) and paste0("utm_", size).
NEW BUG
timeseries_ndvi.py — dimensions and seasonal formula still don't match R and Julia
benchmarks/timeseries_ndvi.py
▼

The fix now correctly computes NDVI from red/NIR bands, but two fundamental mismatches remain:

# Python generates: 46 dates × 250×250 pixels
n_dates, height, width = 46, 250, 250

# R and Julia generate: 12 dates × 500×500 pixels
n_dates = 12; width = height = 500

# Python seasonal formula
red = 0.1 + 0.2 * (1 - base_vegetation * base_vegetation)  ← squares base_vegetation

# R/Julia seasonal formula
red = 0.1 + 0.2 * (1 - base_vegetation * vegetation_level)  ← uses vegetation_level

The dataset size differs by 3×, the vegetation dynamics are computed differently, and there is no vegetation_level seasonal term in Python. Mean NDVI, trend, and amplitude statistics will all differ substantially. Cross-language hash comparison will never pass.

Fix
Change Python to n_dates=12, height=500, width=500 and replace the vegetation formula with vegetation_level = 0.5 + 0.3 * sin(2π*(t)/12) then compute bands as R/Julia do.
